Apesar do HTML5 já resolver os problemas de validação de dados no lado cliente, o javascript ainda é bastante utilizado para essa finalidade. Você deverá criar uma pagina com os seguintes campos: -Nome Completo -Obrigatório - Mínimo de 6 caracteres - Deve conter pelo menos dois nomes - Email - Obrigatório - Deve aceitar o formato [nome@dominio] - Site - Deve aceitar somente URLs válidas - Assunto - Tipo select com as opções: - Dúvida - Reclamação - Elogio -Como ficou sabendo da Pizza Mama Mia? - Tipo checkbox com as opções: - Internet - Amigos - Pesquisa (Google, Yahoo, Bing, etc) - Outros - Sexo - Masculino - Feminino - Mensagem - Obrigatório - Mínimo de 10 caracteres - Máximo de 140 caracteres - Deve exibir um contador de caracteres Pra cada campo desse deve ser exibido uma mensagem diferente para cada um. [Exemplo de formulário](http://media.smashingmagazine.com/wp-content/uploads/2011/03/wufoo-form-validation1.png) ### Importante: - Você **não pode** utilizar jQuery para fazer essa atividade. - Seu código HTML deve ser validado com o **doctype HTML5** - Seu código CSS deve ser validado com a **versão 3** - Sua aplicação **deve funcionar** no Chrome, Firefox e Internet Explorer - O design da página de formulário deve ser com **base na logo** da [Pizza Mama Mia](http://logopond.com/gallery/detail/222284)
